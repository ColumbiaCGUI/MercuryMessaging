**Static Analysis (main work):**

For EACH tutorial 1-12, perform the following and write results into PRE_ANALYSIS.md:

A. **Fetch the wiki page** using WebFetch or by reading the cloned wiki repo markdown file (e.g., `Tutorial-1-Introduction.md`). Extract every code block from the wiki.

B. **Read the actual tutorial scripts** from `Assets/MercuryMessaging/Examples/Tutorials/TutorialN/TN_*.cs`. For tutorials with subdirectories (e.g., Tutorial4/Scripts/), read those too.

C. **Compare each wiki code block** against the corresponding actual script. For each code block, document:
   - Wiki class name vs actual script class name (e.g., wiki says `ChildResponder`, script is `T1_ChildResponder`)
   - Wiki method signatures vs actual method signatures
   - Wiki base class vs actual base class
   - Wiki `using` statements vs actual
   - Any API calls in wiki that don't exist in the actual code
   - Any deprecated Unity APIs used (check for `Shader.Find("Standard")`, `Physics.autoSyncTransforms`, legacy VR APIs)

D. **Document keyboard controls** listed in the wiki and map them to the corresponding `Input.GetKeyDown()` calls in the actual scripts. Flag any key mentioned in wiki but not in code (or vice versa).

E. **Document expected console output** from the wiki and map to `Debug.Log()` calls in scripts. Flag mismatches in log message text.

F. **Rate the tutorial health**: GREEN (wiki matches code), YELLOW (minor naming differences only), RED (significant logic/API/behavior mismatches).
